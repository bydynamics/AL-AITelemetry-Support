# Managing Saved Queries

## Saving a query

1. After creating or generating a query, open the **Telemetry Query Card**.
2. Give the query a descriptive **Name**.
3. Optionally add a **Description** to explain what the query does.
4. The query is automatically saved.

## Organising queries

- Use meaningful names that describe the purpose (e.g., *"Daily Error Summary"*,
  *"Slow Pages Last 7 Days"*).
- The query list supports searching and filtering.

## Editing a saved query

1. Open the query from the **Telemetry Queries** list.
2. Modify the KQL in the editor.
3. Changes are saved automatically when you leave the card.

## Deleting a query

1. Select the query in the **Telemetry Queries** list.
2. Choose **Delete** from the action bar.
3. Confirm the deletion.
